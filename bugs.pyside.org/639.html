Description Hugo Parente Lima 2011-01-24 17:45:04 EET 
In Qt bug 16462 comments, a qdoc3 developer told that the WebXML generator
among other experimental generators will be removed from Qt.

http://bugreports.qt.nokia.com/browse/QTBUG-16462?page=com.atlassian.jira.plugin.system.issuetabpanels:comment-tabpanel&focusedCommentId=139859#comment-139859

Nowadays PySide uses the XML output of qdoc3 WebXML generator to create the
PySide documentation, we need to find another solution like Doxygen, but we
don't known if Doxygen can correct parse all tags used in Qt documentation.
Comment 1 Matti Airas 2011-01-25 05:44:15 EET 
Hugo, thanks a lot for catching this one.

I'm a bit undecided whether this should be P3 or P4. Assuming WebXML isn't
removed by the time we release 1.0, this could be P4, but in that case PySide
couldn't be built with later Qt versions. Making P3 - comment, if you want it
changed...
Comment 2 Hugo Parente Lima 2011-02-16 11:34:28 EET 
Matti, I'm lowering the priority to P4 until qdoc3 WebXML generator being
removed from Qt, so we can fork qdoc3 only when necessary.

About the problem pointed by you that this PySide version *docs* wouldn't be
built with future versions of Qt isn't really a problem since we always specify
the Qt version supported by each PySide release.

Is it ok for you?
